# System Overview

AttenLink is a QR-code-based attendance management system designed to simplify the attendance process for both teachers and students. The system consists of two main parts: a frontend application that provides the user interface and user experience, and a backend application that handles business logic, authentication, attendance processing, and data management. Together, these components create a complete digital attendance workflow that replaces manual roll calls with a faster and more structured solution.

The main purpose of AttenLink is to allow teachers to create and manage sections, open attendance sessions, generate QR codes for student check-in, and review attendance reports. At the same time, students can log in, join sections using a code, scan QR codes during active sessions, and review their attendance results. The system is designed to support a simple classroom workflow while remaining flexible enough for future expansion.

The frontend is responsible for presenting the system to users in a clear and interactive way. It provides interfaces for login, section management, attendance session management, QR code scanning, and attendance review. Teachers use the frontend to create sections, manage sessions, generate QR codes, and monitor attendance records. Students use the frontend to join sections, scan QR codes, and view their personal attendance status. In this way, the frontend acts as the communication layer between users and the core system.

The backend is responsible for processing requests from the frontend and enforcing the rules of the system. It provides RESTful APIs for authentication, section management, session management, QR code generation, enrollment, attendance check-in, and reporting. The backend also manages data validation, authorization, token verification, and database operations. By separating business logic from the user interface, the backend ensures that the system remains secure, maintainable, and scalable.

From a workflow perspective, the system begins when a teacher logs in and creates a section for a class. Inside that section, the teacher can create one or more attendance sessions for specific meetings. When attendance begins, the teacher generates a QR code linked to the active session. Students who are enrolled in the section can scan the QR code through the frontend, and their attendance will be validated and recorded by the backend. After the check-in process is complete, the teacher can view attendance records and generate reports. Students can also review their attendance results from their own view.

The system follows a client-server architecture. The frontend acts as the client, sending requests and displaying responses, while the backend acts as the server, providing APIs and managing the database. This separation makes the system easier to maintain because interface changes on the frontend can be handled independently from the business logic on the backend. It also supports future improvements such as mobile applications, role-based permissions, real-time attendance tracking, and advanced reporting features.

In summary, AttenLink is a complete attendance management system that combines a user-friendly frontend with a structured backend. The frontend focuses on usability and interaction, while the backend focuses on logic, security, and data consistency. Together, they provide an efficient platform for managing attendance through QR-code technology.
